# End-of-Contract Data Handling

Document what happens to customer and personal data when a contract, tenant, or service relationship ends.

## Template

- trigger event
- data returned to controller or customer
- data deleted
- deletion deadline
- residual retained data and legal basis
- backups and archival treatment
- subprocessor follow-up actions
- verification evidence
- owner

## Minimum Expectations

- return and deletion paths are explicit
- tenant offboarding is operationally repeatable
- backups and archives are not ignored
- subprocessors receive matching offboarding instructions where required
