# Client Customization Policy

Define how customer-specific changes are handled without collapsing the shared product into one-off forks.

## Template

- customization request
- shared product fit
- tenant-configurable option
- feature-flagged option
- dedicated custom implementation
- operational cost impact
- maintenance owner
- decision outcome

## Working Rules

- prefer configurable shared behavior over forks
- isolate exceptions explicitly when a custom path is unavoidable
- document support and upgrade impact before accepting custom work
