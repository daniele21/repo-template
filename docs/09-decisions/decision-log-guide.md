# Decision Log Guide

Use ADRs for architectural or operational decisions that materially affect the project.

Create or update an ADR when:

- changing major boundaries
- choosing a core dependency or platform
- introducing a new integration pattern
- accepting a significant tradeoff or workaround

Each ADR should capture:

- context
- decision
- alternatives considered
- tradeoffs
- consequences
