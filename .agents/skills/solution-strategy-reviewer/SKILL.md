# Solution Strategy Reviewer

Use this skill when brainstorming has produced multiple realistic approaches and the repo needs a recommended direction before planning or coding.

## Inputs

- project brief
- brainstorming doc
- repository context
- constraints
- user priorities

## Workflow

1. Compare only realistic strategies worth considering.
2. Evaluate architecture, stack, cost, maintainability, delivery speed, compliance, and lock-in implications.
3. Make tradeoffs explicit without pretending the choice is obvious when it is not.
4. Recommend one strategy and state which tradeoffs are accepted by that choice.

## Output

- strategies being compared
- strengths and weaknesses of each strategy
- architecture implications
- cost implications
- privacy / GDPR implications
- AI governance / AI Act implications when relevant
- security implications
- delivery implications
- recommended strategy
- accepted tradeoffs

## Review Questions

- Which strategy best fits the stated priorities and constraints?
- What becomes easier or harder with each option?
- Which tradeoffs are acceptable now and which would create future debt?
