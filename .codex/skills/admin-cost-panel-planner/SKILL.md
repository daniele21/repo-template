# Admin Cost Panel Planner

Use this skill to define how administrators should monitor product costs and usage.

## Inputs

- project brief
- solution strategy
- delivery plan
- architecture and observability context

## Workflow

1. Identify the main cost drivers and the operational decisions they influence.
2. Define the minimum telemetry needed for useful admin visibility.
3. Recommend attribution dimensions such as tenant, provider, workflow, or feature.
4. Propose thresholds, alerts, or anomaly views when they support real operational action.
5. Keep privacy and security implications of monitoring data explicit.

## Output

- main cost drivers
- minimum telemetry required
- recommended admin-facing metrics
- attribution dimensions
- recommended thresholds or alerts
- suggested admin panel structure
- data-collection considerations
- privacy and security considerations for monitoring data

## Review Questions

- Which metrics support real product or operational decisions?
- Where is cost attribution needed to prevent blind spots?
- What monitoring detail is worth the implementation and privacy cost?
