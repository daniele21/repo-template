# Project Plan Orchestrator

Use this skill when the project is moving from approved strategy to delivery planning.

## Inputs

- approved solution strategy
- project brief
- brainstorming doc
- repository context

## Workflow

1. Break the work into incremental slices or milestones.
2. Surface dependencies and cross-cutting workstreams.
3. Define testing, documentation, compliance, security, UX, and observability work that must travel with delivery.
4. Keep the plan aligned with the approved strategy and avoid silently changing architectural intent.

## Output

- implementation sequence
- vertical slices or milestones
- dependencies between slices
- testing requirements
- documentation requirements
- privacy / GDPR tasks
- AI governance / AI Act tasks when relevant
- security tasks
- UX / UI tasks
- observability and admin cost monitoring tasks
- definition of ready
- definition of done

## Review Questions

- What is the smallest slice that proves progress without violating the strategy?
- Which workstreams must be cross-cutting instead of deferred?
- What must be true before implementation can start safely?
